กิจกรรมที่ 10 : API endpoint
คําสั่ง
1. ใหกลุมรวมกันเขียน หรือ Capture หนา UI ของ Application ของตนเอง โดยเปนหนาที่เปนกิจกรรมหลัก
หนาที่เปน Input ขอมูลพื้นฐาน หรือ Register หรือ Login ไมตองทํา
2. ใหกลุมรวมกันกําหนด endpoint
3. ใหกําหนดรูปแบบ JSON ที่สง
4. เขียน sequence diagram ในขอ 1 จนครบโดยเรียงตามความซับซอนของกิจกรรม มาก -> นอย
5. สรางขอมูลพื้นฐานสําหรับแสดงผล และ นําใสเขาใน object ตางๆ
6. เขียน method ตาม API endpoint ใหทํางานไดสมบูรณ
7. นําไปเชื่อมกับ FastAPI และทดสอบโดย Swagger หรือ Postman
การสงงาน
- เปนงานกลุม ดังนั้นตองมีอยางนอย 4 API โดยแตละคนตองรับผิดชอบ
- ใหแสดงหนา UI ที่จะพัฒนาตอไป
- ใหแสดงรูปแบบ JSON ที่สงกลับ โดยตองสอดคลองกับ UI
- ใหแสดง Sequence Diagram
- ใหรัน API ใหดู เพื่อทดสอบวาไดขอมูลที่ควรจะเปนหรือไม
- โปรแกรมที่เขียน ตองสอดคลองกับ Sequence Diagram
